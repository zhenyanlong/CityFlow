# TDD 文档更新与同步

此 skill 用于在实现完一个功能模块后，根据聊天上下文和实际代码/资产内容更新技术设计文档，并保持中英文版本同步。

## 适用场景

- 用户说 "更新 TDD" 或 "更新设计文档"
- 用户完成了一个功能模块的实现，需要将实际实现情况记录到 TDD
- 代码实现与 TDD 设计有偏差，需要将文档对齐到实际实现

## 工作流程

### 1. 收集信息

在更新 TDD 之前，需要收集以下信息：

- **聊天上下文**：回顾本次会话中用户实现的功能点、讨论的设计决策、遇到的问题和解决方案
- **代码变更**：使用 `grep` / `SearchCodebase` 等工具查看本次实现涉及的核心文件，了解：
  - 新增/修改了哪些类、函数、接口
  - 数据结构和算法是否有变化
  - 模块之间的交互方式是否有调整
- **资产变更**：查看是否新增或修改了 Blueprint、材质、网格等资产

### 2. 对比现有 TDD

- 阅读 `TDD.md` 中与本次实现相关的章节
- 找出以下差异：
  - TDD 中描述的设计 vs 实际实现的代码
  - TDD 中缺失的新功能点
  - TDD 中已经过时的描述

### 3. 更新 TDD.md

**必须遵守的原则：**

- **增量修改**：只修改与本次实现相关的章节，不要重写整个文档
- **以实际代码为准**：TDD 描述必须与已实现代码保持一致
- **保持格式**：沿用 TDD.md 现有的 Markdown 格式规范（标题层级、表格、代码块、列表等）
- **记录差异**：如果实现有意偏离了原始设计，在对应描述中简要说明原因（可选）

**常见更新类型：**

| 类型 | 操作 |
|------|------|
| 新增模块/类 | 在对应章节添加新的子节，描述职责和关键接口 |
| 修改数据结构 | 更新对应的代码块，修改字段和注释 |
| 流程变更 | 更新步骤列表，调整顺序或增减步骤 |
| 性能/限制调整 | 更新第 3 章和第 4 章的相关内容 |

### 4. 同步到 TDD_Chinese.md

更新完 `TDD.md` 后，**必须**同步到 `TDD_Chinese.md`。参考 `tdd-sync` skill 的工作方式，以更新后的 `TDD.md` 为源，逐段翻译变更部分到中文版。

- 如果只有少量章节变更，只翻译变更的章节
- 翻译时保持与 `tdd-sync` skill 相同的规则

### 5. 汇报变更

以简洁的列表形式向用户汇报本次更新了哪些内容：

```
TDD 更新摘要：
- 新增 xxx 模块描述
- 更新 yyy 数据结构
- 修改 zzz 流程步骤
```
