# TDD 文档同步

此 skill 用于保持 `TDD.md`（英文）和 `TDD_Chinese.md`（中文）两份技术设计文档的内容一致。

## 适用场景

- 用户明确要求 "同步 TDD 文档"
- 修改了 `TDD.md` 或 `TDD_Chinese.md` 之后，需要将改动翻译并同步到另一份文档
- 两份文档内容不一致，需要对齐

## 工作流程

### 1. 确定同步方向

首先阅读两份文档，判断哪一份是"源"（最近被修改过的、内容更新的一方），哪一份是"目标"（需要被覆盖更新的一方）。

- 如果用户明确指示了方向，按用户指示执行。
- 如果没有明确指示，比较两份文档的内容和修改时间，以内容更完整、更新的一方为源。

### 2. 逐段翻译并同步

采用**逐段**方式，将源文档的内容翻译后写入目标文档，而非一次性写入整个文件。这样做的好处是：
- 可以精确控制每段翻译的准确性
- 便于在翻译过程中发现问题及时修正
- 保持中英文段落结构一一对应

翻译规则：
- 章节标题、表格、代码块、列表等 Markdown 结构保持与源文档一致
- 代码中的类名、函数名、变量名（如 `GridManager`, `ARoadTile`, `ConnectedMask` 等）保持原文不翻译
- 技术术语使用业界通用译法
- 源为英文时，目标文档使用中文；源为中文时，目标文档使用英文

### 3. 验证

同步完成后，快速对比两份文档的：
- 章节数量是否一致
- 表格行数和列数是否对应
- 代码块是否匹配
