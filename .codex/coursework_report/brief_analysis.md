# Coursework Brief Analysis

## Key submission facts

- Module: WM9M6-30 Fundamentals of Games Research, Development and Management
- Assessment: Game project
- Weight: 60%
- Deadline: 22 June 2026, 12:00 UK time
- Submission: Tabula
- Documentation format: one document, 3-4 pages, using the supplied LaTeX template
- Maximum assessment word count: 3,600 words equivalent, with a 10% allowance before a penalty applies
- AI level: AI Collaboration. AI may support drafting and review, but the student must critically check and edit the work.

## Required game application evidence

The submitted game should:

1. Be fully playable.
2. Use at least one key feature from the previous critical review.
3. Make that feature central to the gameplay or visuals.
4. Include an interactive tutorial or in-game help.
5. Include player feedback such as UI cues and sound effects.

CityFlow currently addresses these requirements through its road-planning game loop, data-driven tutorial, localised UI, placement feedback, traffic alerts, evaluation report, and audio settings.

## Required report sections

The final 3-4 page report must contain:

- Short GDD: core mechanics, win/loss or scoring, engagement, narrative, and art style.
- Short TDD: architecture, game loop, scene management, object-oriented structure, key systems, and tools/libraries.
- Technical overview: engine, languages, and target platform.
- Critical feature justification: identify the feature from the critical review and explain its effect on graphics, gameplay depth, and engagement.
- Testing and debugging summary: testing methods, bugs found, and evidence of iterative development.

## Confirmed critical-review link

The previous critical review studied **procedural content generation (PCG) of roads**, with a focus on **L-systems**. The report therefore treats CityFlow's assisted road generator as the critical feature. The implementation is not a direct textbook L-system. It is a hybrid system that combines organic rule-based growth with a reserved connection plan, attraction towards building entrances, budget limits, and validation against one shared road component.
